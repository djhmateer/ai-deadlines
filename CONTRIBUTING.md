# Contributing to Red:4 Store

We welcome contributions. Here's how to help.

## Getting Started

1. Fork the repository
2. Clone your fork
3. Install dependencies: `pnpm install`
4. Create a branch: `git checkout -b your-feature`

## Before Submitting

- Run tests: `pnpm test`
- Follow the style guides in `CLAUDE.md`
- One PR per feature or fix
- Write descriptive commit messages

## Code Style

- 2 spaces, no tabs
- ES modules (`import`/`export`)
- One assertion per test
- No LLM-generated business logic (see `CLAUDE.md`)

## Pull Request Process

1. Update tests if needed
2. Ensure tests pass
3. Submit PR with clear description
4. Respond to review feedback

## Reporting Bugs

Open an issue with:
- What you expected
- What happened
- Steps to reproduce

## Questions?

Open an issue. We'll respond within a few days.
