# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability, **do not open a public issue**.

Email us at: **security@red4aerospace.com**

Include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested fixes (optional)

## Response Timeline

- **Acknowledgment**: Within 48 hours
- **Initial assessment**: Within 1 week
- **Resolution target**: Depends on severity

## Scope

This policy covers:
- The Red:4 Store API
- Authentication and authorization
- Data handling and storage
- Payment processing integration

## Recognition

We're happy to credit security researchers who report valid vulnerabilities responsibly. Let us know how you'd like to be acknowledged.

## Supported Versions

| Version | Supported |
|---------|-----------|
| main branch | ✅ |
| Older releases | ❌ |

We only patch the latest version. Update frequently.
